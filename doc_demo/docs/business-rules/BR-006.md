## Rule Statement

> A detail record is only processed if it is associated with the current authorization summary record.

## Condition

When a detail record is read.

## Action

The program checks if the detail record's key matches the current summary record's key. If they don't match, the detail record is skipped, and the program proceeds to find the next detail record.
